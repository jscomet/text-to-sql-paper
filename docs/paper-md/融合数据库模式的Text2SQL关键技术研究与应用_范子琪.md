# 摘 要

Text2SQL 是自然语言处理科研领域中的一项重要任务,在研究智能问答系统中起着 关键性的作用,其核心任务是从自然语言中抽取出用户的查询意图,并将其转换成 SQL 语句,从而使用户可以通过自然语言查询数据库中的信息。Text2SQL 的未来发展前景 广阔,它可以改变传统的数据库查询方式,使之容易使用,更好满足用户需求。

当前研究重点为提高 SQL 子句任务的匹配准确率,但忽略了 SQL 的句法生成的正 确性,涉及多表连接的 SQL 生成仍存在大量错误。论文的主要研究内容及成果如下:

针对复杂数据库模式下的 Text2SQL 模型方案在多表连接问题上准确率偏低,通过 引入逆规范化技术,构建一种新的 Text2SQL 方法 RNSQL(逆规范化网络)。该方法使 用逆规范化对数据库模式优化重构,提取用户查询中的关键词,将其与逆规范化后的数 据库模式链接,提高模型的整体性能;该方法还关注 SQL 句法生成的正确性,在 SQL 句法生成阶段引入了一系列的限制和约束,确保生成的 SQL 语句符合 SQL 语法规则。 通过逆规范化方法,对数据库模式进行重构,从而削减数据库中模式边界,进一步用分 解关联查询的方式来生成查询。重构后的数据库模式通过神经网络"编码-解码"与语 义信息模式链接。逆规范化的优势不仅体现在减少连接操作的需求、减少数据库模式中 索引和外键的数量,更是加快查询 SQL 生成的速度,经理论分析和在公共数据集 Spider 上实验验证,RNSQL 相较于改进前模型精确匹配度上升 3%,能有效提升 Text2SQL 任 务的质量。

针对基于逆规范化的 Text2SQL 模型在模式优化重构时模式质量不稳定的问题,构 建一种基于模式分解的 Text2SQL 方法,该方法通过模式分解评估优化逆规范化模式链 接,链接自然语言语句与数据库重构模式,关注模式重构高效化,命名 RNSDE(逆规 范化模式分解评估生成网络)。在链接自然语言语句与重构模式时,采用基于特征的匹 配方法,将自然语言语句中的特征与重构模式中的特征进行匹配,匹配结果确定了重构 模式、连接方式与 SQL 语句的生成,并采用一种并行组件化的算法实现模式重构高效 化,从而提高 SQL 语句的生成速度,优化 SQL 语句生成质量,经理论分析和在公共数 据集 Spider 上实验验证,RNSDE 相较于改进前模型精确匹配度上升 3.7%,能有效提升 Text2SQL 任务的质量。模型应用于专题数据库上,成果转化一套民族地区深度贫困专 题智能问答系统。

关键词:语义解析; Text2SQL; 逆规范化; 槽填充; 数据库模式感知

Text2SQL is an important task in the field of natural language processing scientific research and plays a key role in the study of intelligent question-and-answer systems. Its core task is to extract the user's query intent from natural language and convert it into SQL statements so that the user can query information in the database through natural language. Text2SQL has a promising future, it can change the traditional. The future development of Text2SQL is promising, it can change the traditional database query method, make it easy to use and better meet the user needs.

The current research focuses on improving the matching accuracy of SQL clause tasks but ignores the correctness of the syntax generation of SQL, and there are still a lot of errors in SQL generation involving multiple table joins. The main research contents and results of the paper are asfollows:

A new Text2SQL method RNSQL is constructed by introducing inverse normalization techniques for the Text2SQL model scheme in complex database schemas with low accuracy on multi-table join problems. The method optimally reconstructs the database schema using inverse normalization, extracts the keywords in user queries and links them to the inverse normalized database schema to improve the overallperformance of the model;the method also focuses on the correctness of SQL syntax generation and introduces a series of restrictions and constraints in the SQL syntax generation stage to ensure that the generated SQL statements conform to the SQL syntax rules. Through the inverse normalization method, the database schema isreconstructed to cut down the schema boundaries in the database and further generate queries by decomposing associative queries.The reconstructed database schema is linked to the semantic information schema through the neural network "encoding-decoding". The advantages of inverse normalization are not only in reducing the need for joint operations and the number of indexes and foreign keysin the database schema but also in accelerating the speed of query SQL generation. Theoretical analysis and experimental verification on the public dataset Spider show that the exact matching degree of RNSQL increases by 3% compared with the modelbefore improvement, which can effectively improve the quality of Text2SQL tasks.

To address the problem of unstable pattern quality of the inverse normalization-based Text2SQL model in pattern-optimized reconstruction, a pattern decomposition-based Text2SQL method is constructed, which optimizes inverse normalization pattern linking by pattern decomposition evaluation, links natural language statements with database

reconstruction patterns and focuses on efficient pattern reconstruction, named RNSDE. In linking natural language statements and reconstructed schemas, a feature-based matching method is used to match features in natural language statements with features in reconstructed schemas, and the matching results determine the reconstructed schemas, connection methods and SQL statement generation, and a parallel componentized algorithm is used to achieve schema reconstruction efficiency, thus improving the speed of SQL statement generation and optimizing the quality of SQL statement generation, after Theoretical analysis and experimental validation on the public datasetSpider show that RNSDE increases the exact matching degree by 3.7% compared with the modelbefore improvement, which can effectively improve the quality of Text2SQL tasks. The model is applied to the thematic database, and the results are transformed into a set of intelligent question and answer systems on the topic of deep poverty in ethnic areas.

# 第 1 章 绪论

# 1.1 研究背景及意义

结构化查询语言(SQL)是被广泛应用于数据库的查询语言。Text2SQL 是一 种新兴的自然语言处理技术,它可以将自然语言查询转换为 SQL 查询语句,从而 有效地支持用户查询关系数据库中的信息。将自然语言自动解析为 SQL 具有巨大 的潜力,便于非专业用户使用自然语言查询和挖掘结构化数据。

Text2SQL 是关系数据库智能问答系统的核心[1],也是自然语言处理领域中极 其重要的一项任务。Text2SQL 系统可以大大简化用户与关系型数据库交互的方式, 使得用户只需要使用自然语言描述查询需求,而不需要熟悉 SQL 语言的语法和结 构。为了实现实用的 Text2SQL 工作流,模型需要将自然语言查询与给定的数据 库关联起来。自然语言与关系数据库或表交互是一个重要且具有挑战性的问题。

Text2SQL 的研究可以追溯到 20 世纪 70 年代的自然语言查询系统研究。随着 深度学习技术的发展,Text2SQL 在近几年得到了长足的发展。目前深度学习模型 已经可以将自然语言查询转化为 SQL 查询,并在一定程度上理解 SQL 查询的语 义,提高 Text2SQL 的准确性与泛化性。

Text2SQL 技术的研究主要来源于两个方面:一是以自然语言查询为基础的信 息检索技术;二是数据库技术。在以自然语言查询为基础的信息检索技术方面, Text2SQL 技术借鉴了传统的自然语言处理技术,包括语义分析、句法分析、语言 模型等,以及自然语言查询转换技术,如模板匹配、语义解析等,以有效地支持 用户查询关系数据库中的信息。另一方面,Text2SQL 技术还借鉴了数据库技术, 如 SQL 查询语句的构建、数据库模式分析等,以实现将自然语言查询转换为 SQL 查询语句的目的。Text2SQL 包含两项子任务,首先要构建正确的 SQL 语句框架, 可称为句法生成任务,在此基础上,再将框架中的每一个子元素与相应的值相匹 配,可称为值匹配任务。句法生成任务与值匹配任务并不是完全独立的任务,句 法任务与值匹配任务相互影响相互促进,从而得到高质量 SQL 语句。

如图 1.1 多表单轮 Text2SQL 问题示例所示,单轮 Text2SQL 任务以单句自然 语句(Query)"List all singer names in concerts in year 2014."、数据库模式(Schema) 为输入,得到一条 SQL 查询语句为输出,其中"SELECT"、"FROM"、"JOIN"、 "WHERE"构成了 SQL 语句的句法结构部分,"T2.name"是 SQL 语句中列名 匹配值,"singer\_in\_concert"是 SQL 语句中表名匹配值,其余部分为条件匹配 值。

![](_page_9_Figure_1.jpeg)

图 1.1 多表单轮 Text2SQL 问题示例

其中,句法问题致力于构造合适的槽填充结构承托字符串匹配、启发式融合 编码,以支持复杂本文转化为 SQL 语句模型。在 Text2SQL 中,填充槽算法可以 帮助我们识别问句中的关键字,并将其转换为 SQL 查询语句中的列名、表名、操 作符和值。填充槽算法首先提取问句中的关键字和实体。例如,对于以下问句, "What are names of the top three products in sales volume?",关键字可以是"sales volume"、"top three"和"names",实体可以是"products"。其次将关键字 和实体映射到数据库中的列名、表名和值。对于上述问句,关键字"sales volume" 可以映射到名为"sales"的列,关键字"商品名称"可以映射到名为"product\_name" 的列,实体"products"可以映射到名为"product"的表。最后,组合关键字和 实体,并生成 SQL 查询语句,生成如下的 SQL 查询语句:"SELECT product\_name FROM product ORDER BY sales DESC LIMIT 3";其中,"SELECT"表示要查 询的列名,"FROM"表示要查询的表名,"ORDER BY"表示根据哪个列排序, "DESC"表示降序排序,"LIMIT"表示查询结果的数量。

槽填充方法可以简化 Text2SQL 的实现过程,提高查询的准确度和效率。然 而,它也存在一定局限性,例如无法处理复杂的条件查询和多个表之间的关系, 需要进一步优化和改进。在主流方案中,RYANSQL [2]将句法问题转化为具体的嵌 套句式问题,通过标注语句位置信息将嵌套 SQL 语句转换为非嵌套 SQL 语句, 以此改进草图槽填充方法的准确性,然而表连接的选择错误仍大量存在。

多表单轮问题与单表单轮问题存在巨大任务量差异是多表连接错误的一大 原因,如列名所属表名的选择、多表连接的路径选择等一系列相关任务。在 Text2SQL 中,单表单轮问题和多表单轮问题是两种不同的查询类型,它们的处理 方式和难度也有所不同。单表单轮问题通常是指查询只涉及一个数据库表的问题。 例如,"查询销售量最高的商品名称"就是单表单轮问题中的一个例子。对于这 类问题,Text2SQL 需要识别用户的查询意图和问句中的关键字,然后将其映射为 对应的 SQL 查询语句。由于只涉及一个表,所以查询语句的维度相对较低,

Text2SQL 的实现比较容易。相比之下,多表单轮问题涉及多个数据库表之间的关 系,它的难度要高于单表单轮问题。例如,"查询销售量最高的商品所在的门店 名称"就是多表单轮问题中的一个例子。在这种问题中,用户需要进行多轮查询, 并且需要识别不同的数据库表、表之间的关系和 SQL 查询语句的拼接方式。与单 表单轮问题相比,多表单轮问题需要更加复杂的技术和算法来实现。总体来说, 单表单轮问题和多表单轮问题在 Text2SQL 中的应用有所不同。单表单轮问题相 对来说比较简单,可用的技术和算法较为成熟,而多表单轮问题则需要更加高级 和复杂的技术和算法来解决。与多表单轮 Text2SQL 任务相比,RYANSQL、 RATSQL [3]等主流模型在单表单轮 Text2SQL 任务上普遍具有更高的准确率。

为优化句法结构中表连接问题,本文对句法问题在 Text2SQL 任务中的作用 进行了深入探究。在 Text2SQL 任务中,句法问题具有重要的作用,首先是隐含 信息提取,句法分析可以帮助识别问句中的隐含信息,例如省略的主语、宾语、 状语等,以及多个名词之间相互关联的语法结构。这些隐含信息对于进一步生成 SQL 查询语句非常重要。其次是语义特征提取,句法分析可以提取问句中的语义 特征,例如名词、动词、形容词等。这些语义特征可以作为进一步解析 SQL 查询 语句的依据,帮助系统更准确地识别查询的意图。再次是语法错误检测,在用户 输入自然语言问句时,常常会出现语法错误和歧义,这对 Text2SQL 任务有非常 大的影响。句法分析可以帮助检测语法错误和歧义,从而提高问句理解和语义解 析的准确度和效率。最后是查询语句生成:在将自然语言问句转化为 SQL 查询语 句时,句法分析可以提供语法模板和语法结构,以及关键字的位置、类型等信息。 这些信息对于生成准确的 SQL 查询语句非常重要。

综上所述,句法问题在 Text2SQL 任务中可以提高问句的理解和语义解析能 力,同时也是生成准确的 SQL 查询语句的重要依据。如 Dong [4]等人所提,从模 型的权限中检测和删除特定于域的单词可以让模型专注于学习自然语言和 SQL 之间的语法转换,从而减少输入的句法稀疏性。融合数据库模式感知技术的句法 改进有益于 Text2SQL 任务的生成,模式感知通过挖掘数据库模式中的信息,如 表名、列名、主键、外键,融合模式感知的表连接有助于优化多表连接过程中表 连接路径的选择,也对过渡连接表的模式信息干扰起到一定保护性作用。

# 1.2 国内外研究现状

Text2SQL 在深度神经网络上的应用主要依赖于"编码-解码"框架。"编码- 解码"框架是自然语言处理中的一个经典框架,它在 Text2SQL 任务中也有着重 要的应用。该框架的基本思想是将自然语言输入和目标输出分别编码和解码,从 而实现自然语言理解和生成的过程。

"编码-解码"框架中的编码部分的主要任务是将输入序列转化成向量,即将

自然语言问句及数据库模式信息转化成定长向量;而"编码-解码"框架中的解码 部分,其任务则是将生成向量转化成输出序列,即将定长向量转化成生成的 SQL 查询语句。"编码-解码"框架应用广泛,在发展早期被应用于机器翻译、文本生 成等领域,继而应用在文档摘取、问答系统等领域。在编码器的选择上,编码器 发展经历了基于长短期记忆网络(LSTM)[4]的编码器、基于图神经网络(GNN) [6]的编码器以及基于 BERT 的编码器等编码器技术演变。解码器的研究主要围绕 序列生成,槽填充以及语法生成三种方式。

#### 1.2.1 基于序列生成的 Text2SQL 方法

序列生成方法相对于机器学习方法,具有更好的可扩展性和泛化性。它可以 处理复杂的自然语言问句,例如包含多个嵌套子查询等情况。通过生成 SQL 语句 中的每个组成部分,实现从自然语言问句到 SQL 查询语句的转换过程。其基本思 路是使用循环神经网络(RNN)[7]将自然语言问句编码成一个向量,使用另一个 RNN 根据编码向量逐步生成 SQL 查询语句。在编码阶段对自然语言进行编码, 编码器通常采用一个双向 RNN,将自然语言问句转化为一个向量,这个向量包含 了问句中所有的信息和上下文,它是整个模型的输入。在解码阶段根据编码向量 逐步生成 SQL 查询语句,解码器通常采用一个带有注意力机制的 RNN 来逐步生 成 SQL 查询语句,在每一步中,解码器会利用已经生成的 SQL 组成部分以及询 问向量,生成下一个 SQL 组成部分。最后,生成器会负责将已生成的 SQL 组成 部分组合成一条完整的 SQL 查询语句。但是,序列生成方法也存在问题,包括生 成过程中的歧义和复杂度问题。为了解决这些问题,最近的研究者们也提出了基 于预先定义的模板或任务特定的语法,自然语言生成 SQL 查询语句等方法,来增 强模型的生成精度和速度。

该方法基于神经网络技术,将自然语言语句转换为 SQL 查询的序列,从而避 免了繁琐的规则或模板设计。序列生成模型包括了循环神经网络、长短时记忆网 络和 Transformer 等。近年来,基于序列生成的 Text2SQL 已经得到了快速的发展。 一些优秀的模型,如 SQLNet、Seq2SQL、Editing-based SQL Generation(EditSQL) 等,已经在自然语言和 SQL 之间的转换问题上取得了优异的性能。其中,SQLNet [8] 通过先生成查询的执行过程,再生成查询语句的方式,大大提高了 Text2SQL 的 精度和效率。TypeSQL [9]模型利用类型信息对自然语言中的实体和数字进行标注, 标注的类型作为模型训练中的辅助信。Seq2SQL [10]从输入的自然语言问题中提取 重要信息,并将其组成 SQL 查询序列。Spider 和 EditSQL [11]引入上下文信息来解 决自然语言和 SQL 之间的深层次语义差异。EditSQL 提出了一种基于知识图谱的 智能问答方法,利用 BiLSTM-CRF 模型来实现实体识别。

虽然基于序列生成的 Text2SQL 在自然语言和 SQL 自动转换方面取得进展, 但仍然存在一些问题。其中包括对复杂查询和长文本的处理、命名实体的处理、

正确性和可解释性等。使用基于递归神经网络的编码器-解码器框架进行语义分析 的端到端方法有所增加。这种方法在许多任务中取得了很大的成功,如机器翻译 [13],句法分析,图像字幕等。当翻译模型与 SQL 的语法规则深度结合时,模型的 准确性和训练效率可以得到极大的提高[14]。在语义解析的情况下,它包括学习从 自然语言表达到意义表示的映射,避免了对中间表示的需要。这种灵活性有优点 也有缺点——它减少了定义词汇、模板和手动生成特征的需要,使得模型可以跨 领域和意义表示语言进行推广。然而,传统方法能够更好地建模和利用逻辑组合 的内置知识。基于序列生成的 Text2SQL 研究在语言和数据库领域中具有广泛的 应用价值,其未来的发展将涉及更多的深度学习架构和更多的上下文信息的结合。

#### 1.2.2 基于槽填充的 Text2SQL 方法

基于槽填充的 Text2SQL 方法是一种常见的 Text2SQL 方法,在自然语言问句 中提取出所有的槽位(Slot),并且把每个槽位填入 SQL 查询语句中相应的位置。 这种方法可以避免繁琐的规则或模板设计,并且为复杂的自然语言查询提供了更 多的灵活性。通常情况下,一个槽位可以对应 SQL 语句中的列名、表名、操作符 和值等。首先,对自然语言问句进行词法分析和语法分析,以提取出相关的语法 信息;接着,利用 NLU(自然语言理解)组件将自然语言问句解析成相关的槽位 和其对应的值;最后,生成 SQL 方案,将槽位和值填入特定的 SQL 查询语句模 板中。例如,对于自然语言问句"Show me the best selling books of the month", 经过分析得出其中的槽位为:"书籍类别"和"时间市场",对应 SQL 查询语句 模板可类似如下:SELECT \* FROM books WHERE books.category = {书籍类别} AND books.time = {时间市场} ORDER BY books.sales DESC LIMIT 10;在上述 SQL 查询语句模板中,槽位与 SQL 查询语句相应的位置相关联,通过将识别出的 槽位填入相应的位置,最终生成的 SQL 查询语句可以满足用户的需求。与基于序 列生成方法相比,基于槽填充的 Text2SQL 方法更加规范化和可解释,其生成的 SQL 查询语句较为容易被人类理解和解释。然而,基于槽填充的 Text2SQL 方法 也存在一些问题,例如处理复杂查询的效率低,同时也容易被其它因素(如上下 文)的影响而产生错误答案。

RYANSQL 提出一种基于草图的槽填充方法来生成 SQL 语句,将嵌套的 SQL 查询转换为一组非嵌套的 SELECT 语句,用于复杂的 Text2SQL 问题。RYANSQL 提出了复杂 SELECT 语句预测的详细草图,以及处理嵌套查询的语句位置代码。 此外,还提出两种输入操作方法进一步提高系统整体性能。在具有挑战性的 Spider 基准数据集上实现了最先进的性能。误差分析表明,应该根据其他槽的预测结果 来更新槽值。

Huilin [15]等人结合草图的方法来优化 SQL 生成。基于 SQL 查询的结构,将模 型分解为三个子模块,并为每个子模块设计特定的深度神经网络。我们采用双向

关注机制和卷积神经网络(CNN)的字符级嵌入来改善结果,在 WikiSQL 数据集 中取得了先进的结果。F-SQL [16]采用门机制来融合表模式和表内容,并得到关于 表模式的更多不同的表述,在 WikiSQL 和 TableQA[17]上取得了新的一流的结果。

基于草图的槽填充方法,首先由 SQLNet 提出,用于处理 WikiSQL 数据集, 使用用于 SQL 查询的时隙定义草图,解码器对这些时隙的值进行分类。Hwang [18] 等人和 He 等人[19]在 BERT 的帮助下,在 WikiSQL 数据集上实现了基于草图的方 法最先进的性能。然而,与基于语法的方法相比,基于更复杂的 Spider 基准的草 图方法的性能相对较低。有两个主要原因:(1)很难为 Spider [20]查询定义草图, 因为 Spider 中 SQL 查询允许的语法远比 WikiSQL 查询复杂。(2)由于基于槽填 充的方法会为预定义的插槽填充值,因此这些方法在预测嵌套查询时存在困难。

#### 1.2.3 基于模式链接的 Text2SQL 方法

基于模式链接的 Text2SQL 方法是一种基于自然语言问句和数据库模式之间 的匹配关系来生成 SQL 查询语句的 Text2SQL 方法。该方法通常利用文本和结构 特征来生成一组候选的查询模板,然后将自然语言问句和每个查询模板进行匹配, 从而生成最终的 SQL 查询语句。这种方法可以自动处理一些自然语言查询中的歧 义和不完整之处,并且可以更好地处理复杂的查询。首先建立数据库的结构、实 体和关系等元信息,梳理数据库的元数据,并形成数据库的模式。其次将自然语 言问句进行语法分析、意图识别和实体识别等,并抽取出对应的查询条件、SQL 操作符、SQL 函数等。之后基于数据库的模式信息,利用自然语言问句中的关键 词生成候选模板,例如定义表之间的关系、从表中查询数据等规则。这些模板可 以根据与模式匹配性来进行排序和过滤,选出一组最匹配的备选模板。最后将自 然语言问句和备选模板进行匹配,基于语义和结构特征得分计算来评估模板的匹 配度。最终,选择得分最高的模板对应的 SQL 查询语句作为答案。相比于传统的 基于规则和基于序列生成的 Text2SQL 方法,基于模式链接的 Text2SQL 方法实现 简单,具有较高的可扩展性和泛化性。然而,该方法需要对模式进行解析和处理, 因此在数据量大和数据结构复杂的情况下具有一定局限。

模式链接的引入是新的研究重点之一,如知识图谱领域的实体链接[21-23]和对 话系统中的缓慢填充[24-25],其中提出了大量注释数据和模型,以解决其特定属性。 在语义分析的一般领域,已经证明将底层结构与词汇解耦有利于跨领域语义分析 [26]。然而,当涉及到文本到 SQL 问题时,现有方法常将模式链接视为使用简单启 发式的次要预处理过程,例如自然语言语句和列或表名称之间的字符串匹配[27]。 简单的启发式方法很难准确识别自然语言话语中涉及的列或表,也很难理解话语 与相应数据库模式之间的关系。因此,将图式链接作为一个单独的研究问题迈出 了第一步。然而,由于缺乏直接的模式链接监督,在具有挑战性的 Spider 数据集 上实现了有限的改进,进一步说明了这个问题的困难。与之前的方法不同,将模

式链接集成为可学习的方法。

基于模式链接的 Text2SQL 已经得到了快速的发展。Chen [28]等人制定了两个 链接过程来应对模式链接,将显式自然语言链接到数据库,以及结构链接,将输 出 SQL 中的实体与其在数据库模式中的结构关系联系起来。这两个链接过程的效 果会根据正在生成的实体而变化,使用门控机制在它们之间动态选择。Guo [29]等 人在表内容与问题字符串中的某些单词匹配以及某些表头也与问题字符串中的某 些单词匹配问题上,为深度模型编码了两个加法特征向量。Lei [30]等人构建了一个 简单的基于 BERT 的基线,称为模式链接 SQL(SLSQL)来执行数据驱动的研究。 Long [31]等人通过链接问题和表格内容建立特征向量,然后在预训练的 BERT 基础 上建立分类模型进行微调,并将 Text2SQL 过程分解为多个基于模板填充的多分 类子过程,使用一般分类模型和条件值分类模型来生成 SQL。

虽然基于模式链接的 Text2SQL 已经取得了一些显著进展,但仍存在一些问 题,例如处理复杂的查询、容易受到表格结构的更改和可解释性等。

# 1.3 论文主要研究内容

国内外研究显示,当前研究重点为提高 SQL 子句任务的匹配准确率,但忽略 了 SQL 的句法生成的正确性,涉及多表连接的 SQL 生成质量仍存在巨大改进空 间,需要通过合理的算法设计和评估方法解决。本文针对以上问题,提出了两种 融合数据库模式的 Text2SQL 模型构建方法,并落实应用于专题数据库,具体研 究内容如下:

- (1)针对复杂数据库模式下的 Text2SQL 模型方案在多表连接问题上准确率 偏低,本文构建一种基于逆规范化的 Text2SQL 方法,通过引入逆规范化技术, 对数据库模式优化重构,关注 SQL 句法生成的正确性,命名 RNSQL(逆规范化 网络),提取用户查询中的关键词,并将其映射到逆规范化后的数据库模式上。 在 SQL 句法生成阶段引入了一系列的限制和约束,确保生成的 SQL 语句符合 SQL 语法规则,提高模型的整体性能。
- (2)针对基于逆规范化的 Text2SQL 模型在模式优化重构时模式质量不稳定 的问题,构建一种基于模式分解的 Text2SQL 方法,该方法通过模式分解评估优 化逆规范化模式链接,链接自然语言语句与数据库重构模式,关注模式重构高效 化,命名 RNSDE(逆规范化模式分解评估生成网络)。在分解过程中构建了一个 层次结构,选择不同细粒度的模式和连接方式生成 SQL 语句。在链接自然语言语 句与重构模式时,采用基于特征的匹配方法,将自然语言语句中的特征与重构模 式中的特征进行匹配,匹配结果确定了重构模式、连接方式与 SQL 语句的生成, 并采用一种并行组件化的算法实现模式重构高效化,从而提高 SQL 语句的生成速 度,优化 SQL 语句生成质量,应用于专题数据库上,成果转化一套民族地区深度

贫困专题智能问答系统。

# 第 2 章 Text2SQL 相关技术

Text2SQL 技术近年来受到越来越多的关注和研究,本章针对 Text2SQL 相关 技术进行描述,主要分基于规则的语义解析、编码-解码框架、Text2SQL 优化与 关系模式规范化技术四个部分。

## 2.1基于规则的语义解析

语义分析(Semantic Parsing)作为自然语言处理领域的基础工作之一,涵盖了 语言学、计算语言学、机器学习和知识语言等众多领域,在近年来也得到了普遍 重视。

语义解析早期处理主要是基于规则的,这使得它们具有特定的领域。基于规 则的语义解析是一种常用的自然语言处理技术,它通过预先定义规则来分析自然 语言文本,从而获得文本中的语义信息。

基于规则的解析系统通常由两部分构成:一是规则库,解析规则通常为上下 文无关文法;二是同义词库,记录标准词的同义词。解析是上下文无关文法归约 的过程。首先进行自动分词,接着将问句词依照同义词库归约为标准词,再将词 归约后的问句与规则库中的解析规则比对,比对成功即该条用户问句被成功归约 到该条解析规则所对应的标准问句上。

LUNAR[32]采用基于语法规则,该模型使用语法分析器生成使用规则映射到 底层数据库查询语言的分析树。在语义语法系统中使用了一种更严格的语法,它 考虑了语义类别而不仅仅是句法类别,构建解析树,将非终端节点设置为语义类 别,语义类别约束对树的可能分配,使得从树到查询语言的映射更加容易。虽然 简化给定任务流程,但包含特定领域的语义知识使得适应其他领域更加困难。

基于规则的语义解析方法还可以基于领域专家对领域知识的理解和规则的 描述来实现,易于理解和实现[33]。通过预先定义和优化规则,可以提高其准确性 和可靠性,尤其是在特定领域的文本分析中表现优异。输入的自然语言文本较复 杂时,基于规则的语义解析方法可能无法覆盖所有情况,需要预先定义的规则太 多,实际操作中难度较大。

基于规则的语义解析虽然在特定领域的文本分析中表现优异,但受限于预定 义规则和难以应对复杂场景的特点,在处理较为复杂和多样化的自然语言文本时, 其效果并不理想。本文结合机器学习和深度学习等技术,进一步提高语义解析的 准确性和效率。

# 2.2编码-解码框架

深度学习背景下,Text2SQL 任务可视为类似神经机器翻译的一种序列到序列

的生成任务,主要采用 Seq2Seq 模型框架。

在编码阶段,自然语言问题与数据库模式之间需要形成准确的映射关系,例 如自然语言问题涵括了哪些数据库表格中的哪些模式信息(数据库列名和数据库 表格值);在问题与数据库模式映射关系准确的情况下,也要准确映射问题与 SQL 语句语法,具体来说就是问题词对应哪些 SQL 语句关键词操作,例如聚合、排序、 选择、分组等;除此之外,自然语言语句表达中涉及的逻辑结构需要正确表示并 转化为生成的 SQL 查询语句,逻辑结构转化中难点聚焦在嵌套、多子句等。在解 码阶段,SQL 查询语言是需要严格依据其逻辑结构进行构造,保证 SQL 语句的语 法正确与执行成功。

在 Text2SQL 任务中,编码器可以将自然语言问句中的关键信息和语义特征 转化为向量表示,同时解码器可以根据这些向量生成对应的 SQL 查询语句。编码 器将自然语言问句转化为向量表示。这可以通过词嵌入技术和循环神经网络 (RNN)实现。编码器可以将问句中的关键信息和语义特征提取出来,转换为向 量表示,这些向量包括问题的编码、上下文编码等。解码器将向量表示转化为 SQL 查询语句。解码器可以根据编码器生成的向量表示,生成与之对应的 SQL 查询语 句。如图 2.1 自然语言处理中的 Encoder-Decoder 框架所示,在此过程中,解码 器通过逐步解码出每个部分的 SQL 查询语句,例如关系、聚合和选择结果等。

对于多表查询等复杂问题,可采用 Attention 机制,通过注意力分配的方式将 查询中关键信息对应到相应的表或列上,进一步提升模型的精度和效率。"编码- 解码"框架可以帮助 Text2SQL 任务实现自然语言到 SQL 查询语句的转化,同时 可以处理多表查询等复杂问题。该框架广泛应用于各类自然语言处理任务[34],带 来了不少的成功案例,目前研究人员正致力于进一步提高模型的准确率和泛化性 能。

#### 2.2.1 Bert 模型

Bert [35]模型是一种预训练语言模型,被认为是自然语言处理领域的一大突破。 Bert 模型是基于 Transformer Encoder 架构的深度双向神经网络,可以在各种自然 语言处理任务中达到最先进的水平。与传统的语言模型相比,Bert 模型在设计上

具有两个显著的特点:一是采用了 Masked Language Model(MLM)和 Next Sentence Prediction(NSP)两种训练方式,能够更好地处理双向信息和语义关系; 二是采用了多层 Transformer Encoder 结构,使得模型具有更深的学习能力,可以 学习长范围依赖关系。

图 2.2 Bert 模型下游任务

在训练过程中,Bert 模型首先通过 MLM 方法随机遮盖输入文本中的部分词, 通过 Transformer Encoder 结构进行双向计算,学习词向量的上下文表征。同时, Bert 模型还采用了 NSP 方法来训练模型对句子级别的任务进行推断,使得模型具 有更强的语言理解和语义关系判断能力。经过预训练后,Bert 模型可以直接应用 于不同的自然语言处理任务[36-38],如文本分类、序列标注、文本生成等,从而为 自然语言处理领域的研究和应用带来了重大的进展和挑战。

Arora [39]等人受到 BERT 风格的预训练方法的启发,研究了使用基于注意力的 模型来学习关系数据库中实体的嵌入,解决了在由多个表组成的关系数据库上学

BRIDGE [40]有效地捕获了所需的跨模态依赖关系,用于在跨数据库语义解析 中对自然语言问题和关系数据库之间的依赖关系进行建模,通过 BERT 中微调的 深度关注实现。

#### 2.2.2 注意力机制

注意力机制(Attention)用于选择性地关注输入序列中的某些部分,从而提 高任务的准确性和效率。Attention 的基本思想是在给定的输入序列中,根据任务 的需要,赋予不同的权重值,以便更有效地处理和编码信息。

注意力机制是一种神经网络模型中的重要组成部分,其功能是通过对输入数 据进行选择性过滤,仅保留相关信息,从而提高模型在处理复杂任务时的效率和 准确性。具体来说,注意力机制通常分为两种类型:硬注意力和软注意力。硬注 意力的思路是将输入数据中最相关的部分进行选择和聚焦,其他部分则被忽略。 而软注意力则更加灵活,它考虑到不同部分的相关程度不同,可以根据具体情况 对不同部分赋予不同的权重,从而更为精确地进行选择性过滤。

#### 2.2.3 Transformer 模型

Transformer [43]是一种广泛应用于自然语言处理领域的神经网络架构,由 Google 团队在 2017 年提出并应用到了机器翻译任务上。它基于 Self-Attention 机 制,可以更有效地处理长文本序列,并且无需使用循环卷积结构,因此可以并行 计算,大大缩短了训练时间。

Transformer 架构主要由两个部分组成:Encoder 和 Decoder。Encoder 用于将 输入的源语言文本序列编码成一个上下文向量,提供给 Decoder 进行目标语言文 本的生成。Encoder 和 Decoder 都由多个堆叠的注意力机制和前馈神经网络组成, 其中注意力机制可以帮助模型关注不同部分的输入序列。

在 Transformer 的注意力机制中,自注意力机制被广泛使用。通过自注意力机 制,模型可以在序列中的每个位置同时计算出注意力权重,使得模型能够更好地 解析句子中各个单词的含义和关系,从而提升模型的翻译效果。Transformer 模型 已经被应用于各种自然语言处理任务上,例如文本分类、情感分析、机器翻译、 问答系统等,具有良好的性能和广泛的适用性。

#### 2.2.4 公路网络

Highway Network(公路网络),是一种在神经网络架构中应用的技术,旨在解决梯度弥散与过拟合等问题。它最初由 Schmidhuber<sup>[45]</sup>等人在 2015 年提出,并被广泛应用于自然语言处理等领域。公路网络主要由两个部分组成:传统的前馈神经网络和一个贯穿整个网络的转换门(Transform Gate)。其中,前馈神经网络用于提取输入数据的特征,而转换门则用于控制前馈信号和跳过信号(即输入信号直接到达输出),使得信息在不同层之间能够流通。神经网络的深度对模型效果有很大的作用,可是传统的神经网络随着深度的增加,训练越来越困难,公路网络的出现解决了深层网络难以训练的问题,通过门控机制来实现,通过门控来组成"信息通道"。一层神经网络的表示用H来表示一个非线性转换,参数为 $W_H$ ,当输入为X时,输出为 $Y = H(X, W_H)$ 。

# 2.3 Text2SQL 优化

随着自然语言处理和数据库技术的发展,Text2SQL 解决方案也得到了不断的 演变和改进。最早的 Text2SQL 方法是使用手动编写的规则来将自然语言查询转 化为 SQL 语句,这种方法能够实现简单的查询,但不能很好地处理复杂的自然语 言语句。为了改进规则匹配方法的不足,研究者们提出了许多针对缺陷的处理方 法,例如对于省略的信息进行预测和补全等。

近些年来基于神经网络的方法表现突出,成为该领域的主流。基于 Seq2Set 的改进方向使用约束模板来将自然语言转化为 SQL 语句。模板来自于已经存在的 SQL 语句模板库,模板的填充则可以根据自然语言查询语句来完成。这种方法因 其高效性而广泛应用。基于 TRANX 的改进方向对 SQL 语句进行语法分析来生成 SQL 语句,这种方法能够大大提高转化的精确性和稳定性,但需要预先编写语法 规则。

#### 2.3.1 基于指针网络的优化

指针网络(Pointer Network)是一种神经网络结构,它能够模拟出循环神经 网络的能力,同时又能够直接从输入序列中选择最适合的内容进行输出。在 Text2SQL 任务中,指针网络可以针对输入中的某些关键信息,选择最优的 SQL 语句中的关键词进行填充。这种方法避免了比较常见的基于标签的方法,即预训 练 SQL 代码中的标签,以及基于规则的方法,即仅针对某些预定义的查询进行匹 配。与其他神经网络语言模型相比,指针网络更加擅长处理树形结构、序列排序 等方面的复杂问题,并可提高概率估计的准确性。因此,在 Text2SQL 及相关任 务中,指针网络技术的使用可以提升数据表述和 SQL 语句生成的质量和效率。

SQL 组成来自三部分:数据库中元素(如表名、列名、表格元素值)、问题 中词汇、SQL 关键字。其次,当前公开的多领域数据集为了验证模型数据库无关, 在划分训练集和测试集时要求数据库无交叉,这种划分方式导致测试集数据库中 很大比例的元素属于未登录词。传统的 Seq2Seq 模型是解决不好这类问题的。

由于指针网络可以较好的满足具体数据库无关这一要求,在多领域数据集上 的模型大多使用该网络,如 Seq2SQL、Coarse2Fine [46]、IRNet 等模型。

本文应用指针网络较好解决这一问题,其输出所用到的词表是随输入而变化 的。具体做法是利用注意力机制,直接从输入序列中选取单词作为输出。在

Text2SQL 任务中,将问题中词汇、SQL 关键词、对应数据库的所有元素作为输入 序列,利用指针网络从输入序列中拷贝单词作为最终生成 SQL 的组成元素。

#### 2.3.2 基于 Seq2Set 的优化

基于 Seq2Set 的改进技术是一种性能突出的方法。Seq2Set 技术是一种基于编 码器-解码器框架的方法,用于解决输入序列与输出集合之间的映射问题。其核心 思想是将输入序列编码为一个固定大小的向量,然后通过解码器生成一个输出集 合。在 Text2SQL 任务中,输入序列为自然语言,输出集合为 SQL 语句中的关键 字集合。使用 Seq2Set 技术,可以将 Text2SQL 中的复杂语法结构转化为更简单的 指令或子任务,例如查询、选择等子任务。

在简单问题对应的数据集合上,其 SQL 查询语句形式简单(仅包含 Select 和 Where 关键词),为了解决 Seq2Seq 模型中顺序错误带来的影响(如"条件 1 And 条件 2",预测为"条件 2 And 条件 1",属于顺序错误,但对应的 SQL 是正确 的),SQLNet 提出了 Seq2Set 模型,基于所有的列预测其属于哪个关键词(即属 于 Select 还是 Where,在 SQLNet 模型中仅预测是否属于 Where),针对 SQL 中 每一个关键词选择概率最高的前 K 个列。该模式适用于 SQL 形式简单的数据集, 在 WikiSQL 和 Text2SQL 数据集合上使用较多,且衍生出很多相关模型,如 TypeSQL、SQLova、X-SQL [47]等。

本文应用这种方法的核心是将自然语言句子中的关键词提取出来,并进行分 类和调整。该技术的另一个优点是,它可以在生成过程中处理输入中的任意长度 的语法和结构,同时避免了生成冗余和无效的语句。Seq2Set 技术适用于 Text2SQL 任务,可以有效地提升任务的性能和效率,并能够处理大量的自然语言表述,以 生成准确有用的 SQL 语句。

#### 2.3.3 基于 TRANX 的优化

TRANX(Transformation-based Neural Abstract Syntax Parser),是一种基于 转换的神经抽象语法分析器,被广泛应用于 Text2SQL 任务中,以实现更准确的 SQL 语句转换。TRANX 的核心思想是将自然语言输入与完整的 SQL 语句进行分 离,并提取自然语言中的多个语法片段,然后将这些片段转换为 SQL 查询,结果 可以通过构建 SQL 查询语句获取。该技术可以有效解决 Text2SQL 任务中的意向 识别和结构转换问题,并能够自动准确地匹配自然语言查询和 SQL 查询。具体来 说,TRANX 通过一系列的 Transformer 和 LSTM 层,对自然语言查询进行编码, 然后将编码信息进行解码,生成部分或完整的 SQL 查询。TRANX 不仅能够直接 产生 SQL 语句,还可以生成对复杂查询进行解释的自然语言表达。相比于其他基 于模板的 Text2SQL 方法,TRANX 可以根据输入的不同上下文自适应生成 SQL 语句,在多样化和灵活性方面具有优势。这使得 TRANX 在大规模自然语言文本 中的准确率和泛化能力方面都有保证,为自然语言处理减轻了很多负担。TRANX

框架借鉴了 AST 论文思想,根据目标语言的语法构建规约文法,基于该文法可以 将生成目标表示为语法树,需要保证生成目标与语法树表示一一对应,实现了自 顶向下的语法树生成系统。

复杂问题对应的 SQL 查询语句形式也复杂,涉及多关键词组合、嵌套、多子句等。并且,测试集合中的某些 SQL 查询语句形式在训练集合中没有出现,这就要求模型不仅对新数据库具有泛化能力,对新 SQL 查询语句形式也要有泛化能力。

基于语法生成的 Text2SQL 方法是一种使用语法规则和查询模板生成 SQL 查询语句的 Text2SQL 方法。该方法通常基于自然语言问句的语法和结构,利用词法分析、语法分析和语义分析等技术,将自然语言问句转换成具有语法规则的中间表示形式,最后再基于生成的中间表示形式,生成 SQL 查询语句。这种方法的优点在于它可以显式地建模自然语言问句和 SQL 查询语句之间的语法和结构关系,从而提高 Text2SQL 的准确性和可解释性,并且可以支持复杂的查询语句。缺点在于,需要手动编写语法规则和查询模板,这对于复杂的查询场景来说比较困难。同时,在语法规则和查询模板的生成过程中,也容易出现漏洞和错误导致查询结果不准确。为了保证 SQL 生成过程中语法合理,一些模型开始探索及使用语法树生成的方法,如 IRNet<sup>[48]</sup>、GlobalGNN<sup>[49]</sup>、RATSQL、Syntax SQLNET<sup>[50]</sup>。

ValueNet<sup>[51]</sup>模型提出了 ValueNet light 和 ValueNet 两个端到端的 NL-to-SQL 系统,使用具有挑战性的 Spider 数据集来整合数值,主要思想是使用底层数据库的元数据信息,同时使用基础数据的信息作为神经网络架构的输入。

## 2.4 关系模式规范化技术

规范化是一种数据库优化技术,规范化理论提出各范式级别是在分析函数依赖条件下对关系模式分离程度的一种测度,范式级别可以逐级升高。低一级范式的关系模式通过模式分解转化为若干个高一级范式的关系模式的集合,这种分解过程称为关系模式的规范化。

规范化对关系进行投影,消除决定属性不是候选键的任何函数依赖。如图 2.5 规范化理论范式递进过程所示,操作步骤可先对 1NF 关系进行投影,消除原关系中非主属性对码的部分函数依赖,将 1NF 关系转换成若干个 2NF 关系。再对 2NF 关系进行投影,消除原关系中非主属性对码的传递函数依赖,将 2NF 关系转换成若干个 3NF 关系。最后对 3NF 关系进行投影,消除原关系中主属性对码的部分函数依赖和传递函数依赖,也使决定因素都包含一个候选键。得到一组 BCNF 关系。

图 2.5 规范化理论范式递进过程

在数据库设计中,规范化技术旨在消除关系模式中的冗余,去除数据依赖中 不合适的部分,以及解决数据插入和删除时可能出现的异常情况。通过规范化, 可以有效地减少数据冗余,减小数据存储空间,避免数据异常、数据不一致等问 题,提高数据库的查询效率、数据可靠性和维护性。但规范化也会导致查询效率 的下降,需要在规范化和查询效率之间取得平衡。为此,关系数据库的关系模式 必须满足一定的标准,这些标准被称为范式,它们可以根据不同程度的规范化要 求来设定。

## 3.1 RNSQL: 基于逆规范化的 Text2SQL 模型

为深入研究数据库模式在多表连接上的作用,本章提出 RNSQL 模型,模型基于编码-解码框架,模型编码部分如图 3.1 所示,包括一个五层结构输入编码器基本模型,与一个逆规范化模式重构组件(RN),解码部分由表 3.1 子任务与对应填充槽表所示十层填充槽构成。本章在文本特征编码使用了最预训练模型Bert,并迁移和改进了英文上模板填充的处理方式在本文的多表单轮 Text2SQL 任务上,利用深度学习模型构建分类模型,构建填充槽,并针对数据库模式的特点,引入逆规范化与模式链接,优化 Text2SQL 任务解决方案。

#### 3.1.1 RNSQL 优化目标

Text2SQL 任务形式化表示为式(3.1):

$$sql_a = text2SQL(q_i, D) (3.1)$$

输入由自然语言语句数据库模式集合组成,输出表示 SQL 语句,数据库模式集合包含了表名、列名、主键标记、外键关系等信息,2 其中表示数据库中的表名集合,表示数据库中的列名集合,PK表示主键标记集合,FK表示外键关系集合。

本章工作可形式化表示为式(3.2):

$$f = \sum_{\forall q_i \in Q} G(Z_i, Z_i') \tag{3.2}$$

 $D^{c}$ 为逆规范化后的数据库模式, $sql_{q_{i}}$ 为逆规范化后生成的 SQL 语句,RN 为逆规范化过程,其形式化表现为式(3.3):

$$D^{C} = RN(D) \tag{3.3}$$

式 (3.2) 中 G 函数定义如式 (3.4) 所示:

$$G(Z_i, Z_i') = \begin{cases} 1, Z_i' - Z_i > 0 \\ 0, Z_i' - Z_i \le 0 \end{cases}$$
(3.4)

 $Z_i$ '表示逆规范化后综合评估值, $Z_i$ 表示未经逆规范化过程综合评估值,其中,函数 G表示逆规范化后与逆规范化前的 SQL 语句精确准确率之差, $Z_i = EMA(\operatorname{sql}_{q_i})$ ,其中  $SQ_{q_i}$ 表示逆规范化后生成 SQL 语句,形式化表示为式 (3.5):

$$sql_{q_i}^{'} = text2SQL(q_i, D^c)$$
(3.5)

当  $EMA(\operatorname{sql}_{q_i}) > 0$  时,则  $Z_i$  取正值,当  $EMA(\operatorname{sql}_{q_i}) = 0$  时,  $Z_i$  取 0。 EMA 为 SQL 语句准确率评估函数,其中  $\operatorname{sql}_{q_i(\operatorname{gold})}$  为提供的标准 SQL 语句,定义为式(3.6):

$$EMA(sql_{q_{i}}) = \begin{cases} 1, \ sql_{q_{i}} = sql_{q_{i}(gold)} \\ 0, \ sql_{q_{i}}! = sql_{q_{i}(gold)} \end{cases}$$
(3.6)

当  $EMA(\operatorname{sql}_{q_i})=1$ 时,此时逆规范化后的  $\operatorname{SQL}$  语句输出正确,当  $EMA(\operatorname{sql}_{q_i})=0$ 时,此时逆规范化后的  $\operatorname{SQL}$  语句输出错误。本文工作旨在求得使式(3.2)中任意输入函数  $\operatorname{G}$  之和最大,即令  $\operatorname{SQL}$  查询语句生成质量提升最优的  $\operatorname{RN}$  模型。

#### 3.1.2 RNSQL 网络模型

基于逆规范化的 Text2SQL 网络模型基于编码-解码框架,编码部分由嵌入层、编码嵌入层、问题列对齐层、表编码层与问题表对齐层联结,解码部分由依据 SQL 句法设计的十层填充槽构成。

嵌入层引入基于 BERT 的输入编码器,其输入由数据库模式(Database D)、位置信息代码(SPC)与自然语言问题(Q)组成,如式(3.7)所示:

$$[CLS], W_1^Q, [SEP], C_j^P, [SEP], W_K^{C_1}, [SEP], ..., W_K^{C_m}, [SEP]$$
 (3.7)

在BERT编码器输入向量中,[CLS]标志符置于第一个句子的首位,通过BERT编码后得到的表征向量能够使用于后续分类子任务,而[SEP]标志符则用于分开两条输入语句。

嵌入编码层引入 Glove, Glove 是一种用于捕捉语料词汇间的语义特征的词表征工具, Glove 将语料库词汇表示为实数构成的向量表征, 不仅引入词频特征, 更是引入语料库词汇上下文局部特征, 以抓取语料库词汇之间的相似性。

位置信息代码在嵌入编码层由嵌入向量 $\{e_1^P,...,e_p^P\}$ 经过最大池化层得到位置信息代码语义向量 $V^P$ ,将 $V^P$ 与问题及模式信息嵌入向量连接,利用密集卷积神经网络对问题和数据库模式中的每个单词编码,深度卷积层捕获相对位置信息。

在列编码层模型使用 DenseCNN 对问题和列名中的单词进行编码,以捕获语义特征,输出问题词编码  $V^Q=\{v_1^C,...,v_n^Q\}\in R^{n\times d}$  及隐藏列向量  $H^C=\{h_1^C,...,h_m^C\}\in R^{m\times d}$  。

在问题列对齐层,将问题标记与相应的列联系起来,用问题的上下文信息对列向量进行建模。如公式(3.8),用缩放的点积注意将问题标记与列向量对齐。

$$A_{Q2C} = soft \max(\frac{H^c * (V^q)^T}{\sqrt{d}}) * V$$
(3.8)

对于数值较大的缩放因子 d 值,点积呈现数量级的大幅增长,将 softmax 函数推向具有极小梯度的区域。为了减小点积大幅增长带来的影响,对点积做扩展,对 A 做启发式融合,得到融合列矩阵  $F^c$  ,在融合列矩阵  $F^c$  上应用 Transformer,以捕获上下文列信息,输出列编码向量  $V^C = \{v_1^C, ..., v_m^C\}$ 。

# 3.2 基于逆规范化技术的数据库模式重构

逆规范化技术是一个通过添加冗余数据的数据库优化技术,引用逆规范化技 术实施逆规范化模式重构,其目的是是通过降低表和表的链接,进而降低外键及索 引的数量,引入逆规范化技术可以有效提升检索速率。产生的关系越多,则连接操 作就越频繁,则会影响查找速度。

#### 3.2.1 逆规范化技术方案

数据库模式重构的实现基于多种技术,如关系代数、函数依赖、范式理论等。 基于逆规范化技术的数据库模式重构是指对已有的数据库模式进行优化和重构, 以提高查询性能和减少冗余数据的出现。

将冗余的数据添加到一个或多个表中,在关系型数据库中避免冗余的连接。 把一个规范化的模式变成非规范化的过程被称为逆规范化,以提升系统计算速度 的操作[52]。在传统的规范化数据库中,将数据存储在独立的逻辑表中,并试图将 冗余的数据降到最低。使数据库中的每条数据只有一份拷贝。例如,在一个规范 化的数据库中,存在一个课程表和一个教师表。在课程表中的每个条目都会存储 课程的教师 ID,但不包括教师名。当我们需要检索所有带有教师姓名的课程列表 时,我们将在这两个表中做一个连接。

逆规范化技术方案可拆分为四种逆规范化方式,其一为添加冗余列法,其二 为添加派生列法,其三为重新组表法,其四为表分割法。增加冗余列法即在同一 个数据库的多个表中存在相同名称列,这种方法常在查询时避免连接操作。增加 派生列法即将数据库中其他表中的列数据增至当前表中,数据列的选择通常通过 计算得到。增加派生法在查询时可避免跨表查询,同时减少多表连接中 JOIN 操 作的数目。重新组表法即将数据库中的两张表重新组成一张单表以减少连接的次 数从而提高查询性能。表分割法即对表做分割,这种方法能够提高查询性能。表 分割法通常有两种操作方式:其一为水平分割方式,其操作是将数据库表中一列 或多列数据的值进行分割,分割后的数据行分别放入两张独立的表中。其二为垂 直分割方式,其应用适合于表中某些列常用而其余列不常用场景下,其操作是将 主键及部分列合并到一张表中,将主键及剩余列合并到另一张表中。

其主要步骤包括分析数据库模式的结构和属性,确定涉及的表格和属性,进 行逆规范化操作,将多个表格合并为单个表格,在新的表格中建立索引以提高查 询效率,最后对新的表格进行测试和优化。为避免数据的冗余和错误,采用一些 基本的原则,如保持数据的一致性、避免数据的不一致性、使用合适的索引等。 数据库逆规范化模式重构将一个数据库中的表按外键关系分成适当个数的区块, 对每一个区块中的表群做逆合适的规范化操作,使数据库拥有数张没有外键关系 的单表。再将这些单表逐个连接,连成一张单表,完成数据库逆规范化模式重构

操作,投入基础模型。

3.3.2 评价指标

Text2SQL 任务的评价指标主要包括 2 种:精确匹配率(exact matching accuracy) 和执行准确率(execution accuracy)。精确匹配率指预测得到的 SQL 语句与标准 SQL 语句精确匹配成功的问题占比。为了处理由成分顺序带来的匹配错误,当前 精确匹配评估将预测的 SQL 语句和标准 SQL 语句按着 SQL 关键词分成多个子句, 每个子句中的成分表示为集合,当两个子句对应的集合相同则两个子句相同,当 两个 SQL 所有子句相同则两个 SQL 精确匹配成功。为了解决顺序问题所导致的 错误判断,如 WHERE 子句中条件的顺序、投影列的顺序,SQL 语句被拆解成 SELECT、WHERE、GROUPBY 等组件,将每个组件的成分以集合的形式表示, 并依次比较对应集合。例如,对于 SELECT 组件"SELECT avg(col1), max(col2), min(col1)",首先将其表示为集合"(avg, min, col1),(max, col2)",再与标准查询 的集合进行比较。仅当所有组件都与标准查询一致时,预测的查询才是正确的。 执行准确率比较给定数据库中 2 个 SQL 查询的执行结果,当预测查询与标准查询 的执行结果一致时,认为预测查询是正确的。精确匹配率的计算较简单,而执行 准确率的计算则需要数据库实例的支持。本章工作以精确匹配率作为 RNSDE 的 验证指标更为合理与正确。

# 第 4 章 基于模式分解的 Text2SQL 方法研究与应用

数据库模式在多表连接查询问题优化和处理任务上仍有深度可研究空间。在第3章进行多表连接查询时,存在查询结果集包含大量冗余数据的情况,针对该情况通过数据库模式中定义的关系和约束进行优化和处理,从而减少冗余数据的出现。此外,数据库模式可通过定义视图的方式,将多个表连接成一个虚拟的表格,从而简化查询语句、提高查询效率。同时,数据库模式可用于数据的安全性和完整性控制,通过定义访问控制和数据约束等机制来保证数据的安全和完整性。

因此,深入研究数据库模式在多表连接上的作用,为更好理解和利用数据库的数据结构、属性、关系和约束等特性,提高跨表查询的效率和可靠性,也为数据库的设计和优化提供指导。本章基于模式分解的 Text2SQL 方法研究目标是通过将自然语言查询进行基于模式分解评估的逆规范化处理来提高查询性能和准确性。

## 4.1 RNSDE: 基于模式分解评估的 Text2SQL 模型

为优化逆规范化的作用,本章提出 RNSDE(Reversed Normalisation with Schema Decomposition Evaluation)模型,模型基于编码-解码框架,RNSDE 对逆规范化后生成模式使用模式分解评估方法找出最优模式,以对表-模式对齐层进行增强。

#### 4.1.1 RNSDE 优化目标

Text2SOL 任务形式化表示为式(4.1):

$$sql_{q_i} = text2SQL(q_i, D)$$
 (4.1)

输入由自然语言语句及数据库模式集合组成,输出表示 SQL 语句,数据库模式集合包含了表名、列名、主键标记、外键关系等信息,其中表示数据库中的表名集合,表示数据库中的列名集合,PK表示主键标记集合,FK表示外键关系集合。本章工作围绕任务效果提升最大化展开,可形式化表示为式(4.2):

$$f = \max \sum_{\forall q_i \in Q} G(Z_i, Z_i')$$
(4.2)

 $D^L$ 为逆规范化后的数据库模式, $sql_q$ 为逆规范化后生成的 SQL 语句,RN 为逆规范化过程,其形式化表现为式(4.3):

$$D^L = RN(D) \tag{4.3}$$

 $D^{L}$ 为逆规范化后数据库模式集合式,对逆规范化后的数据库模式集合做模式分解评估,得到可用性最高的数据库模式  $D^{m}$  ,其形式化表现为式(4.4):

$$D^{m} = SDE(D^{L})$$
 (4.4)

式(4.2)中 G 函数定义如式(4.5)所示:

$$G(Z_{i}, Z_{i}') = \begin{cases} 1, Z_{i}' - Z_{i} > 0 \\ 0, Z_{i}' - Z_{i} \le 0 \end{cases}$$
(4.5)

 $Z_i$ '表示逆规范化后综合评估值, $Z_i$ 表示未经逆规范化过程综合评估值,其中,函数 G表示逆规范化后与逆规范化前的 SQL 语句精确准确率之差, $Z_i = EMA(sql_{q_i})$ ,其中  $sql_i$ 表示逆规范化后生成的 SQL 语句,形式化表示为式(4.6):

$$sql_{q_i}^{'} = text 2SQL(q_i, D^m)$$
 (4.6)

当  $EMA(sql_{q_i}) > 0$  时,则  $Z_i$  取正值,当  $EMA(sql_{q_i}) = 0$  时,  $Z_i$  取 0。 EMA 为 SQL 语句准确率评估函数,其中  $sql_{q_i(gold)}$  为提供的标准 SQL 语句,定义为式(4.7):

$$EMA(sql_{q_i}) = \begin{cases} 1, \ sql_{q_i} = sql_{q_i(gold)} \\ 0, sql_{q_i}! = sql_{q_i(gold)} \end{cases}$$
(4.7)

当  $EMA(sql_{q_i})=1$ 时,此时逆规范化后的 SQL 语句输出正确,当  $EMA(sql_{q_i})=0$ 时,此时逆规范化后的 SQL 语句输出错误。本文工作旨在求得使式(4.2)中任意输入函数 G之和最大,即令 SQL 查询语句生成质量提升最优的 RNSDE 模型。

模式分解评估模块作用于 RNSDE 模型中问题模式对齐层,经过逆规范化处 理的数据库模式构建新的逆规范化数据库模式集合,在构建集合的基础上,通过 比较不同细颗粒度的模式分解方案,从自然语言句子中学习单词与带有标签的数 据库模式之间的相似性。使用自然语言处理技术对文本语句进行解析,将其转换 为可用于学习的结构化表示形式。然后,采用不同的细粒度度量方法来比较文本 中的单词和数据库模式标签之间的相似程度。最终,构建一个具有高可解释性和 准确性的模型,可以有效地对自然语言语句和数据库模式之间的相似性进行预测 和解释。

#### 4.2.1 逆规范化集合构建

数据库逆规范化模式重构将一个原本存在大量外键关联的数据库表按照它 们的关系进行分组,对每个分组中的表进行逆规范化操作,使得每个分组内的表 都不再存在外键关联。接着,将每个分组最终得到的新表再逐个连接起来,这样 就可以得到一张单一的表,从而完成整个数据库的逆规范化模式重构。这一过程 不仅可以提高数据库的查询效率和可维护性,还能够有效减轻数据库的存储压力, 提升数据库性能和响应速度。

#### 4.2.2 模式分解评估

模式分解评估模块的作用是通过计算保持函数依赖,对数据库模式集合 *D* '中 的模式关系进行分解,使其分解后的关系闭包与标准预测中的闭包并集相等。对 数据库模式集合进行分解,保证无损连接分解,保持函数依赖,减少信息重复。 通过给定正则覆盖中的每个依赖显式地构造一个模式,确保了保持依赖。提取分 解后的闭包与最优预测闭包做模式链接,利用模式分解列表预测 FROM 子句,对 生成模式做预选评估排序。

数据库模式分解算法指的是将一个包含多个关系表的数据库模式分解为两 个或多个较小且较简单的模式的过程,常常用于优化数据库结构以提高查询效率,

应用模式分解算法可以帮助降低数据库冗余,提高数据的存储效率和查询效 率,同时还可以减少数据异常的情况。因此,在具体进行模式分解算法实现过程 中,需要综合考虑数据库模型本身的特点、系统的实际需求以及数据访问的效率 和一致性等方面因素,以保证分解后的数据库模式能够在系统中得到可靠、高效 的应用。在问题语句中,连接表信息鲜少出现,对连接表信息与问题语句关键词 做链接,补充连接表信息,在查询生成中更容易生成正确连接路径,转化为最优 数据库模式。

在逆规范化模式集合中,产生的冗余信息对查询结果影响较大,部分过度冗 余带来的数据重复以及数据错误为数据查询造成干扰,为优化数据库模式选择, 对逆规范化生成数据库模式集合做模式分解评估。首先,通过特定的度量和分类 标准将数据库中的表和关系划分成多个不同的群组,以便于对每个群组进行进一 步的分解;其次,对于每个群组,根据其特定的规范化标准,将其进一步分解为 多个具有独立语义的子模式;最终,将这些子模式逐个连接起来,得到一个全新 的模式,从而实现对原数据库模式的分解。

数据库模式分解算法的核心是基于角色分解、聚合分解、共性分解等方法来 分解数据库模式。

角色分解将数据集分解为多个互相独立且具有特定角色的子系统或子集的 方法。在基于角色分解的算法中,将包含通用字段的表分解成新的单独的表,通 过新的主键字段将这些表之间进行连接。这种方法通常使用矩阵分解等技术来确 定不同的角色或子系统,根据这些角色进行进一步的分析和建模。通过角色分解, 可以更好地理解系统或数据集的不同方面,并使用这些信息来制定更有效的决策 和策略。

通过角色分解既可以提高数据处理效率,又可以保留数据结构和特征,使得 查询范围精准,提高查询准确率。

图 4.3 模式分解算法选择

在聚合分解中,将多个表中的信息聚合在一起,形成一个新的聚合表。这种 方法通常使用聚类等技术来确定子集,让相似的数据被聚合在一起,形成更大的 群体。聚合分解有助于提高数据的处理效率和模型的准确性,同时还可以帮助发 现数据的潜在模式和关联性。

在共性分解中,采用多重继承和分层的方式。这些分解方法可以通过不同的 度量标准和算法模型来实现,并根据具体的需求进行优化和改进,从而获得更高 效的数据库模式分解算法。将数据分解成若干个具有相同特征的部分。在共性分 解过程中,将数据中的共性和差异进行分析,对数据进行分组和分类,以便对数 据进行更精细的分析和处理。模式分解算法的选择基于 Bert,如图 4.3 模式分解 算法选择所示。输入文本首先需要进行处理和编码,Bert 使用的是基于词片(Word piece)的编码方式,把整个文本分成多个词片。之后,Bert 会在文本开头加上一 个特殊标记[CLS],用于表示整个文本的分类结果。每个词片经过嵌入层之后变成 一个向量,称为词片嵌入(Word piece Embedding)。这个过程使用的是一个包含 多个嵌入层的深度神经网络模型。Bert 模型中的最重要的部分是多头自注意力机 制(Multi-Head Self-Attention),它将输入的文本序列转化为一个向量序列,每 个向量代表一种特定的文本信息。基于注意力机制得到的向量序列会经过多个隐 藏层的处理,用于提取和学习更高层次的特征。最后一个隐藏层的输出被用于分 类任务。对于分类问题,通过一个全连接层将每个文本向量映射为一个标量输出,

代表其分类得分。对于多分类问题,同样使用全连接层,但输出层的节点数变为类别数。最终,Bert 会输出所有文本的分类得分,选择最高得分的类别作为最终预测结果,选择最优模式分解算法。

![](_page_42_Figure_2.jpeg)

图 4.4 数据库模式链接感知表示

在逆规范化模式集合上应用模式分解算法,如图 4.4 数据库模式链接感知表示所示,生成备选数据库模式集合  $S\{S_1,S_2,S_3,...,S_m\}$ ,将模式链接分布的基本真值模式表示为 $\Theta$ ,将估计值表示为 $\Theta$ ,将自然语言问句嵌入表示 $W_i$ 与模式元素  $S_j$ 的链接概率表示为  $P_{i,j}$ ,计算方式如式(4.8)所示:

$$p_{i,j} = softma \ \mathbf{x}[MLP(W_i \oplus S_j)] \tag{4.8}$$

其损失函数定义如式(4.9)所示:

$$l = -\frac{1}{|W||S|} \sum_{i=1}^{|Q|} \sum_{j=1}^{|S|} \left[ p_{i,j} \log \overline{p}_{i,j} + (1 - p_{i,j}) \log (1 - \overline{p}_{i,j}) \right]$$
(4.9)

其中  $p_{i,j} \in \Theta$ 是每对  $W_i$ 和  $S_j$ 的基本真值。模式感知表征模式感知的表示由公式 (4.10) 描述:

$$V^{D} = Transformer(f(\ddot{\Theta}, W, S))$$
 (4.10)

其中 f 是参考机制。它的计算方法是通过连接  $W_i$ 的嵌入和模式元素的嵌入  $S_i$ 来计算的。如公式(4.11)所示。

$$f(\ddot{\Theta}, W, S) = \{W_i \oplus \sum_{j=1}^{|S|} \overline{p}_{i,j} W_j\}_{i=1}^{|W|}$$
(4.11)

基于最优模式分解算法选择上,以模式感知学习方式选择最优分解细粒度辅助模式生成,达到更优查询效果。

## 5.1 本文总结

随着数据规模的扩大和计算能力的提高,自然语言处理技术也得到越来越快 的发展,Text2SQL 作为自然语言技术的一项重要应用,也越来越受到科研人员和 商业公司的重视。对 Text2SQL 技术的研究,虽然目前有很多可供借鉴参考相关 领域的研究成果,Text2SQL 领域仍然存在着众多有待解决的困难挑战,因此开展 Text2SQL 领域的相关研究有着重要的价值和广泛的应用前景。

本文对 Text2SQL 任务,以及模式链接的融合上进行了多方面的研究,主要 工作如下:

- (1)针对复杂数据库模式下的 Text2SQL 模型方案在多表连接问题上准确率 偏低,通过引入逆规范化技术,本文构建一种新的 Text2SQL 方法 RNSQL(逆规 范化网络)。该方法使用逆规范化对数据库模式优化重构,提取用户查询中的关 键词,将其与逆规范化后的数据库模式链接,提高模型的整体性能;该方法还关 注 SQL 句法生成的正确性,在 SQL 句法生成阶段引入了一系列的限制和约束, 确保生成的 SQL 语句符合 SQL 语法规则。
- (2)针对基于逆规范化的 Text2SQL 模型在模式优化重构时模式质量不稳定 的问题,构建一种基于模式分解的 Text2SQL 方法,该方法通过模式分解评估优 化逆规范化模式链接,链接自然语言语句与数据库重构模式,关注模式重构高效 化,命名 RNSDE(逆规范化模式分解评估生成网络)。在分解过程中构建了一个 层次结构,选择不同细粒度的模式和连接方式进行生成 SQL 语句。在链接自然语 言语句与重构模式时,采用基于特征的匹配方法,将自然语言语句中的特征与重 构模式中的特征进行匹配,匹配结果确定了重构模式、连接方式与 SQL 语句的生 成,并采用一种并行组件化的算法实现模式重构高效化,从而提高 SQL 语句的生 成速度,优化 SQL 语句生成质量,应用于专题数据库上,成果转化一套民族地区 深度贫困专题智能问答系统。

通过以上工作,本文希望能够给知识库问答工作提供新的思路和方法,并对 基于融合数据库模式及模式链接的形式化查询构建方法的可行性进行探讨和研究。

# 5.2 未来工作展望

本文提出的基于数据库模式的 Text2SQL 方法虽然在主流数据集上取得了不 错的效果,但是由于时间和个人能力的限制,仍然有许多问题和技术可以改进和 完善。

(1)引入逆规范化技术为解码过程带来了更复杂的挑战,模式重构带来的部

分冗余信息对 SQL 语句执行性能带来隐患,后续将进一步研究逆规范化解码过程 的优化以进一步提升模型质量。

- (2)逆规范化重构模式的选择在本文中是决定模式分解细粒度的重点依据, 也是冗余信息干扰程度的重要影响因子,故后续将重点研究逆规范化模式重构的 模式选择优化问题。
- (3)借助领域知识,可以更好地理解自然语言查询,从而更准确地执行数据 库查询。在专题数据库应用中,专业领域知识的理解偏差导致一系列查询错误, 需要更深入研究和开发支持领域知识的 Text2SQL 模型,以便更好地应用于实际 生产环境中。
- (4)在英文查询中,自然语言查询通常不需要分词,直接基于空格和特殊符 号进行匹配。在中文查询中,需先进行准确的分词和词性标注,以便更准确地理 解和解决问题。中文查询中的多义词和语言表达的复杂性使得中文查询的语义理 解更加困难。需要更多研究和实践验证,以便更好将 Text2SQL 方法扩展到中文 数据库。
