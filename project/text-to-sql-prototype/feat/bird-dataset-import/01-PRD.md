## 2. 背景与目标

### 2.1 背景

ICED-2026-paper-code 项目和 Text-to-SQL Prototype 项目都使用 BIRD/Spider 数据集进行 Text-to-SQL 评测，但两者的数据导入方式不同：

- **ICED**: 使用文件系统路径管理 SQLite 数据库，支持命令行导入
- **Prototype**: 使用数据库连接管理系统，通过 `connection_id` 引用连接，需要手动创建连接和任务

用户需要一个简化的方式将 BIRD 数据集导入 Prototype 系统进行评测。

### 2.2 目标

为 Text-to-SQL Prototype 提供标准化的 BIRD 数据集导入功能，使用户能够：

1. 通过 Web 界面上传 BIRD 数据集 zip 文件
2. 或指定服务器本地 BIRD 数据目录
3. 系统自动完成数据解析、连接创建、任务创建
4. 一键开始评测

### 2.3 成功指标

- 用户能在 3 分钟内完成 BIRD 数据集导入
- 支持完整的 BIRD Dev 数据集（11个数据库，1534条问题）
- 数据集格式零转换（直接兼容 ICED 格式）
- 导入成功率 > 95%

## 3. 目标用户

### 3.1 用户角色

#### 角色一：AI 研究人员
- **特征**：从事 Text-to-SQL 模型研究，需要标准化评测
- **痛点**：
  - 已有 ICED 项目的数据集，希望快速在 Prototype 中复现评测
  - 不想手动创建 11 个数据库连接和评测任务
- **需求**：一键导入已有数据，快速开始评测

#### 角色二：数据科学家
- **特征**：需要评估不同 Text-to-SQL 模型性能
- **痛点**：
  - 需要在多个平台上对比评测结果
  - 手动配置评测环境耗时
- **需求**：标准化导入流程，减少配置时间

#### 角色三：开发者
- **特征**：开发 Text-to-SQL 相关功能
- **痛点**：
  - 需要测试评测功能
  - 手动准备测试数据繁琐
- **需求**：快速导入标准数据集进行功能验证

## 4. 需求描述

### 4.1 功能需求

#### FR-001: Zip 文件上传导入
- **描述**: 支持用户通过 Web 界面上传 BIRD 数据集 zip 文件
- **优先级**: P0
- **验收标准**:
  - 支持拖拽上传 zip 文件
  - 支持点击选择 zip 文件
  - 自动解压并解析数据集
  - 验证数据集格式正确性
  - 限制文件大小（最大 1GB）

#### FR-002: 本地目录指定导入
- **描述**: 支持指定服务器本地 BIRD 数据目录导入
- **优先级**: P0
- **验收标准**:
  - 支持输入绝对路径
  - 验证路径存在且可读
  - 验证目录结构符合 BIRD 格式
  - 支持符号链接路径

#### FR-003: 数据集格式验证
- **描述**: 自动验证导入的数据集格式
- **优先级**: P0
- **验收标准**:
  - 检查 dev.json 文件存在
  - 检查 dev_databases/ 目录存在
  - 验证 JSON 格式正确
  - 验证每个 db_id 有对应的数据库文件
  - 提供清晰的错误提示

#### FR-004: 自动创建数据库连接
- **描述**: 为数据集中的每个数据库自动创建连接
- **优先级**: P0
- **验收标准**:
  - 自动提取 db_id 列表
  - 为每个 db_id 创建 SQLite 连接
  - 使用绝对路径避免路径问题
  - 生成连接名称（如"BIRD - california_schools"）
  - 保存 db_id 到 connection_id 的映射

#### FR-005: 自动创建评测任务（父子任务结构）
- **描述**: 使用父子任务结构创建评测任务，父任务作为容器，每个数据库对应一个子任务
- **优先级**: P0
- **验收标准**:
  - 创建一个父任务（`task_type='parent'`）作为容器，存储用户配置
  - 为每个数据库创建一个子任务（`task_type='child'`），通过 `parent_id` 关联父任务
  - 子任务使用对应的 `connection_id` 和 `db_id`
  - 支持选择评测模式（greedy_search/pass_at_k/majority_vote/check_correct）
  - 支持配置 temperature、max_tokens 等参数
  - 子任务继承父任务的配置参数
  - 任务名称格式：父任务"BIRD Dev Dataset"，子任务"BIRD Dev - {db_id}"

#### FR-006: 数据集按 db_id 过滤
- **描述**: 每个子任务只运行对应数据库的问题
- **优先级**: P0
- **验收标准**:
  - 后端自动按 `db_id` 过滤数据集
  - 每个子任务只处理当前 `db_id` 对应的问题
  - 父任务的统计信息自动汇总所有子任务（总问题数、正确数、准确率）
  - 子任务统计信息独立计算
  - 父任务状态根据子任务状态自动更新（全部完成→完成，任一进行中→进行中）

#### FR-007: 导入进度展示
- **描述**: 展示导入过程进度
- **优先级**: P1
- **验收标准**:
  - 显示当前导入步骤（验证→解析→创建连接→创建父任务→创建子任务）
  - 显示已完成/总数进度（如"创建子任务 7/11"）
  - 返回结果包含 `parent_task_id`，便于跳转到父任务详情
  - 显示错误信息
  - 支持取消导入

#### FR-008: 导入结果汇总
- **描述**: 导入完成后展示结果汇总
- **优先级**: P1
- **验收标准**:
  - 显示父任务信息（ID、名称、类型）
  - 显示创建的数据库连接数量
  - 显示创建的子任务数量和列表
  - 显示总问题数量
  - 提供跳转到父任务详情的链接
  - 子任务列表支持点击查看子任务详情

#### FR-009: 父子任务列表展示
- **描述**: 评测任务列表支持层级展示父子任务关系
- **优先级**: P1
- **验收标准**:
  - 父任务行显示任务名称、类型标签、子任务数量、总体进度
  - 支持展开/折叠查看子任务列表
  - 子任务行缩进显示，包含 db_id、状态、进度
  - 任务类型筛选支持 [全部] [父任务] [子任务] [独立任务]
  - 点击父任务详情显示子任务列表表格

#### FR-010: 父任务详情查看
- **描述**: 支持查看父任务详情和子任务列表
- **优先级**: P1
- **验收标准**:
  - 显示父任务基本信息（名称、状态、类型）
  - 显示统计卡片（总问题数、总体正确率、完成进度）
  - 显示子任务状态分布（完成/进行中/等待中/失败）
  - 显示子任务列表表格（db_id、连接名称、状态、问题数、正确率）
  - 支持批量操作（开始全部、重试失败任务）

#### FR-011: 子任务详情查看
- **描述**: 支持查看子任务详情
- **优先级**: P1
- **验收标准**:
  - 显示父任务关联信息（可跳转）
  - 显示数据库连接信息（db_id、连接名称）
  - 显示子任务统计（总问题数、已处理、正确率）
  - 显示评测问题列表（与现有评测详情一致）
  - 支持查看日志和重新评测

### 4.2 非功能需求

#### NFR-001: 安全性
- Zip 文件解压需要验证文件路径，防止路径遍历攻击
- 本地目录导入需要验证路径可读性
- 限制导入目录范围（不允许访问系统目录）
- 子任务数据继承父任务的权限控制

#### NFR-002: 性能
- 导入过程异步执行，不阻塞 Web 界面
- 支持大文件分块上传
- Zip 解压优化，避免内存溢出
- 父任务统计更新使用批量查询优化性能

#### NFR-003: 兼容性
- 支持 BIRD Dev/Train 数据集
- 支持 Spider 数据集
- 兼容 ICED-2026-paper-code 项目数据格式
- 向后兼容现有独立任务（`task_type='single'`）

## 5. 用户界面

### 5.1 评测页面添加导入入口

在评测列表页面添加"导入 BIRD 数据集"按钮，点击后打开导入对话框。

### 5.2 导入对话框

**标签页 1: 上传 Zip 文件**
- 拖拽上传区域
- 文件选择按钮
- 文件信息显示（名称、大小）

**标签页 2: 本地目录**
- 路径输入框
- 路径验证按钮
- 目录信息显示

**公共配置区**
- API Key 选择下拉框
- 评测模式选择
- Temperature 滑块
- Max Tokens 输入

### 5.3 导入进度对话框

- 步骤指示器（验证 → 解析 → 创建连接 → 创建任务）
- 进度条
- 日志输出区域
- 取消按钮

### 5.4 导入结果对话框

- 成功/失败状态
- 统计数据（连接数、任务数、问题数）
- 跳转链接（到评测列表/连接列表）
- 关闭按钮

## 6. 约束与限制

### 6.1 技术约束

- 每个评测任务只支持单个数据库连接
- SQLite 数据库需要使用绝对路径
- 后端服务需要文件系统写权限

### 6.2 业务约束

- 用户需要先配置 LLM API Key
- 数据集需要符合 BIRD 格式规范
- 数据库文件需要可被后端服务访问

## 7. 风险分析

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 路径遍历攻击 | 低 | 高 | 验证文件路径，限制解压目录 |
| 大文件导致内存溢出 | 中 | 高 | 流式解压，限制文件大小 |
| 数据集格式不兼容 | 中 | 中 | 详细的格式验证和错误提示 |
| 权限不足 | 中 | 中 | 提前检查权限，给出明确错误 |
