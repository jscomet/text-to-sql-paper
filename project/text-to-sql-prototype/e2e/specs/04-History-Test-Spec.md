# E2E 测试规范 - 查询历史记录模块

## 模块概述

- **模块名称**: 历史记录 (History)
- **页面路由**: `/history`
- **依赖模块**: Auth, Query
- **优先级**: P1
- **测试用例数**: 12
- **预计耗时**: 2 小时

## 测试环境

### 前置条件

1. 用户已登录
2. 已有查询历史记录（可手动创建或准备测试数据）
3. 至少包含 20+ 条历史记录用于分页测试

### 测试数据准备

```sql
-- 预设历史记录数据
INSERT INTO query_history (user_id, question, sql_text, status, execution_time, created_at, is_favorite) VALUES
(1, '查询所有用户', 'SELECT * FROM users', 'success', 0.5, NOW() - INTERVAL 5 MINUTE, 1),
(1, '统计订单数量', 'SELECT COUNT(*) FROM orders', 'success', 0.3, NOW() - INTERVAL 1 HOUR, 0),
(1, '查找销售额最高的产品', 'SELECT product_name, SUM(amount) FROM sales GROUP BY product_name ORDER BY SUM(amount) DESC LIMIT 1', 'success', 1.2, NOW() - INTERVAL 1 DAY, 1),
(1, '查询不存在的表', 'SELECT * FROM non_existent_table', 'error', 0.1, NOW() - INTERVAL 2 DAY, 0),
(1, '删除所有数据', 'DELETE FROM users', 'blocked', 0, NOW() - INTERVAL 3 DAY, 0);
```

## 测试用例详情

### TC-HIST-001: 历史列表展示

**优先级**: P1
**测试类型**: UI 测试
**对应 UI**: 02-UI-Design.md 3.5节

#### 前置条件
- 用户已登录
- 有查询历史记录

#### 测试步骤

| 步骤 | 操作 | 预期结果 | MCP Playwright 工具 |
|------|------|----------|---------------------|
| 1 | 访问历史记录页 | 显示历史列表 | `browser_navigate` |
| 2 | 验证列表布局 | 显示卡片式列表 | `browser_snapshot` |
| 3 | 验证历史项内容 | 显示问题、SQL预览、元信息 | `browser_snapshot` |
| 4 | 滚动列表 | 加载更多历史 | `browser_evaluate` |

#### 验收标准
- [ ] 以卡片形式展示历史记录
- [ ] 每张卡片显示自然语言问题
- [ ] 显示 SQL 预览（前 2-3 行）
- [ ] 显示数据库连接名称
- [ ] 显示执行时间
- [ ] 显示执行状态（成功/失败）
- [ ] 显示执行耗时

#### 截图记录点
- 历史列表页
- 单条历史卡片详情

### TC-HIST-002: 搜索历史记录

**优先级**: P1
**测试类型**: 功能测试

#### 测试步骤

| 步骤 | 操作 | 预期结果 | MCP Playwright 工具 |

- [ ] 无结果时显示空状态提示
- [ ] 搜索延迟合理（<500ms）

**优先级**: P1
