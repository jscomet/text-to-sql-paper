# E2E 测试规范 - 性能测试

## 概述

性能测试用于验证高级推理功能在高负载和边界条件下的表现，确保系统满足性能指标。

## 测试目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| Pass@K (K=8) | < 15s | 8次采样并行执行总时间 |
| Check-Correct (3轮) | < 30s | 最多3轮迭代总时间 |
| 单次迭代 | < 10s | 生成+检查+修正一轮时间 |
| 并发任务 | >= 4 | 支持并行的推理任务数 |
| 内存占用 | < 2GB | 单次推理任务内存上限 |

## 测试环境

### 硬件要求

- CPU: 4核+
- 内存: 8GB+
- 网络: 稳定连接 LLM 服务

### 软件要求

- 后端服务正常运行
- 数据库连接可用
- LLM API 可访问

## 性能优化建议

### 如果发现性能问题

1. **响应时间过长**
   - 检查 LLM API 响应时间
   - 优化并行执行策略
   - 考虑增加缓存机制

2. **并发性能下降**
   - 检查连接池配置
   - 优化资源竞争
   - 考虑限流机制

3. **内存使用过高**
   - 检查大对象生命周期
   - 优化数据结构
   - 增加内存监控
