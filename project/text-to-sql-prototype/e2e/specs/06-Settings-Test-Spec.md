# E2E 测试规范 - 个人设置模块

## 模块概述

- **模块名称**: 个人设置 (Settings)
- **页面路由**: `/settings`
- **依赖模块**: Auth
- **优先级**: P1
- **测试用例数**: 10
- **预计耗时**: 2 小时

## 测试环境

### 前置条件

1. 用户已登录
2. 有权限访问设置页面

### 测试数据

```typescript
export const settingsTestData = {
  validPassword: {
    current: 'OldPass@123',
    new: 'NewPass@456',
    confirm: 'NewPass@456'
  },
  invalidPassword: {
    short: '123',
    weak: 'password',
    mismatch: {
      new: 'NewPass@456',
      confirm: 'Different@789'
    }
  },
  apiKeys: {
    openai: 'sk-test1234567890abcdef',
    dashscope: 'sk-test0987654321fedcba'
  }
};
```

### TC-SET-001: 查看个人信息

**优先级**: P1
**测试类型**: UI 测试
**对应 UI**: 02-UI-Design.md 3.7节

### TC-SET-010: 表单验证汇总

**优先级**: P1
**测试类型**: 边界测试

#### 验证项

| 字段 | 验证规则 |
|------|----------|
| 用户名 | 必填，2-20字符， alphanumeric |
| 当前密码 | 必填，6-32字符 |
| 新密码 | 必填，6-32字符，含大小写和数字 |
| 确认密码 | 必须与新密码一致 |
| API Key | 可选，格式校验 |
