# E2E 测试规范 - 查询功能模块

## 模块概述

- **模块名称**: 查询功能 (Query)
- **页面路由**: `/query`
- **依赖模块**: Auth, Connection
- **优先级**: P0
- **测试用例数**: 30
- **预计耗时**: 5 小时

## 测试环境

### 前置条件

1. 用户已登录
2. 已配置有效的数据库连接
3. 后端 LLM 服务可用（或已配置 Mock）

### 测试数据准备

```sql
-- 测试表结构
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100),
    age INT,
    department_id INT
);

CREATE TABLE departments (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    location VARCHAR(100)
);

CREATE TABLE orders (
    id INT PRIMARY KEY,
    user_id INT,
    amount DECIMAL(10,2),
    order_date DATE,
    status VARCHAR(20)
);

-- 插入测试数据
INSERT INTO users VALUES
    (1, '张三', 'zhangsan@example.com', 28, 1),
    (2, '李四', 'lisi@example.com', 32, 2),
    (3, '王五', 'wangwu@example.com', 25, 1);

INSERT INTO departments VALUES
    (1, '技术部', '北京'),
    (2, '销售部', '上海');

INSERT INTO orders VALUES
    (1, 1, 1000.00, '2024-01-15', 'completed'),
    (2, 1, 2000.00, '2024-02-20', 'completed'),
    (3, 2, 1500.00, '2024-03-10', 'pending');
```

### TC-QUERY-001: 选择数据库连接

**优先级**: P0
**测试类型**: 功能测试
**对应 UI**: 02-UI-Design.md 3.3节

#### 前置条件
- 用户已登录
- 已创建至少一个连接

#### 测试步骤

| 步骤 | 操作 | 预期结果 | MCP Playwright 工具 |
|------|------|----------|---------------------|
| 1 | 访问查询页面 | 显示查询界面 | `browser_navigate` |
