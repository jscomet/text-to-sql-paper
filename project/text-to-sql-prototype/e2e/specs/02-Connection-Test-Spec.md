# E2E 测试规范 - 数据库连接管理模块

## 模块概述

- **模块名称**: 连接管理 (Connection)
- **页面路由**: `/connections`
- **依赖模块**: Auth
- **优先级**: P0
- **测试用例数**: 20
- **预计耗时**: 3 小时

## 测试环境

### 前置条件

1. 用户已登录
2. 后端数据库服务可用
3. 测试数据库准备就绪

### 测试数据库配置

| 连接名 | 类型 | 主机 | 端口 | 数据库 | 用户名 | 密码 | 预期状态 |
|--------|------|------|------|--------|--------|------|----------|
| MySQL-Valid | MySQL | localhost | 3306 | test_db | root | root | ✅ 在线 |
| PG-Valid | PostgreSQL | localhost | 5432 | test_db | postgres | postgres | ✅ 在线 |
| SQLite-Valid | SQLite | ./test.db | - | - | - | - | ✅ 在线 |
| MySQL-Invalid-Host | MySQL | invalid_host | 3306 | test | root | root | ❌ 离线 |
| MySQL-Invalid-Port | MySQL | localhost | 3307 | test | root | root | ❌ 离线 |
| MySQL-Wrong-Cred | MySQL | localhost | 3306 | test | wrong | wrong | ❌ 离线 |

## 测试用例详情

### TC-CONN-001: 创建 MySQL 连接

**优先级**: P0
**测试类型**: 正向流程
**对应 UI**: 02-UI-Design.md 3.4节

#### 前置条件
- 用户已登录
- MySQL 测试数据库可访问

#### 测试步骤

| 步骤 | 操作 | 预期结果 | MCP Playwright 工具 |
|------|------|----------|---------------------|
| 1 | 登录后访问连接管理页 | 显示连接列表 | `browser_navigate` |
| 2 | 点击"+ 新建连接" | 弹出连接表单弹窗 | `browser_click` |
| 3 | 输入连接名称 "MySQL-Test" | 名称显示 | `browser_type` |
| 4 | 选择数据库类型 "MySQL" | 类型选中，端口自动填充 3306 | `browser_select_option` |
| 5 | 输入主机地址 "localhost" | 地址显示 | `browser_type` |
| 6 | 输入端口 "3306" | 端口显示 | `browser_type` |
| 7 | 输入数据库名 "test_db" | 数据库名显示 | `browser_type` |
| 8 | 输入用户名 "root" | 用户名显示 | `browser_type` |
| 9 | 输入密码 "root" | 密码显示为星号 | `browser_type` |
| 10 | 点击"测试连接" | 显示"连接成功" | `browser_click` |
| 11 | 点击"保存" | 弹窗关闭，列表显示新连接 | `browser_click` |
| 12 | 验证连接卡片 | 显示连接信息和在线状态 | `browser_snapshot` |

#### 验收标准
- [ ] 表单验证通过
- [ ] 测试连接成功提示
- [ ] 保存后出现在连接列表
- [ ] 状态显示为在线（绿点）
- [ ] 显示正确的数据库类型图标

#### 截图记录点
- 步骤 2: 新建连接弹窗
- 步骤 10: 测试成功提示
- 步骤 12: 连接列表
