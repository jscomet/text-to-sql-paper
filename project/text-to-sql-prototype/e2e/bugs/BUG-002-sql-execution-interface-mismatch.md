## 问题描述

SQL 执行成功后，前端无法正确显示结果，报错 `Cannot read properties of undefined (reading 'map')`。

---

## 复现步骤

1. 登录系统，访问查询页面
2. 选择数据库连接
3. 输入问题"查询所有用户的信息"
4. 点击"一键运行"或生成 SQL 后点击"执行"
5. 观察执行结果区域

---

## 期望行为

- SQL 执行成功
- 结果以表格形式展示
- 显示执行时间和行数

---

## 实际行为

- 后端返回执行成功
- 前端报错 `Cannot read properties of undefined (reading 'map')`
- 结果表格无法渲染

---

## 根本原因

前后端接口格式不匹配：

### 后端返回格式
```json
{
  "code": 200,
  "data": {
    "result": {
      "rows": [...],
      "columns": [...]
    }
  }
}
```

### 前端期望格式
```json
{
  "code": 200,
  "data": {
    "rows": [...],
    "columns": [...]
  }
}
```

**问题点**:
1. 后端将 `rows` 和 `columns` 包裹在 `result` 对象中
2. 前端期望 `rows` 和 `columns` 在响应根级别
3. `rows` 格式也不一致：后端返回对象数组，前端期望二维数组

---

## 修复方案

### 后端修复
1. 修改 `QueryExecuteResponse` - 添加前端兼容字段
2. 修改 `QueryRunResponse` - 添加前端兼容字段
3. 修改 `execute_sql_endpoint` - 添加 rows 转换逻辑
4. 修改 `run_query_endpoint` - 添加 rows 转换逻辑

### 前端修复
适配 `result` 对象格式，支持两种响应结构

### 响应拦截器修复
正确处理非 `{code, data}` 格式响应
