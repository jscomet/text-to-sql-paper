### 🟡 TC-EVAL-002: 创建评测任务

**优先级**: P1
**状态**: 🔴 **阻塞** (BUG-004)

#### 执行步骤
1. 点击"新建评测"按钮
2. 填写评测名称
3. 选择数据库连接
4. 选择数据集类型
5. 点击"开始评测"

#### 实际结果
- ✅ "新建评测"对话框正确打开
- ✅ 表单字段完整显示:
  - 评测名称 (输入框)
  - 数据库连接 (下拉选择)
  - 数据集类型 (Spider/BIRD/自定义数据集)
  - 模型 (GPT-4 等)
  - 评测模式 (单模型/多数投票/Pass@K)
  - Temperature (滑块 0-2)
- ✅ 数据库连接下拉正确显示 Test SQLite
- ❌ **创建失败**: 后端返回 422 Unprocessable Entity

#### 错误信息
```
[ERROR] 创建评测任务失败: AxiosError: Request failed with status code 422
```

#### 根本原因
前端发送的 API 请求格式与后端期望不匹配:
- ❌ 缺失 `provider` 字段 (后端必需)
- ❌ `model` 和 `temperature` 嵌套在 `model_config` 中，后端期望平铺字段

详见: [BUG-004 报告](../../bugs/BUG-004-eval-api-mismatch.md)

## 问题记录

### BUG-004: 评测模块 API 请求格式不匹配

**严重级别**: 🔴 Critical
**状态**: 🟡 待修复
**发现时间**: 2026-03-13

#### 问题描述
前端和后端 API 请求格式不匹配，导致创建评测任务时返回 422 错误。

#### 详细信息
- **前端发送**: `model_config: {model, temperature}`
- **后端期望**: 平铺的 `provider`, `model`, `temperature`
- **关键缺失**: `provider` 字段

#### 修复建议
1. 前端：添加 `provider` 字段，展开 `model_config` 为平铺字段
2. 或后端：支持嵌套的 `model_config` 对象

#### 影响范围
- 阻塞 14/15 个评测模块测试用例
- 评测功能完全不可用
