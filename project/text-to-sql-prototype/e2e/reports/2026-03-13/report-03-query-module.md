## 新发现问题

### BUG-002: SQL 执行接口前后端不匹配

**严重级别**: High
**状态**: 🟡 已修复，待验证
**文档**: `bugs/BUG-002-sql-execution-interface-mismatch.md`

#### 问题描述
SQL 执行成功后，前端无法正确显示结果，报错 `Cannot read properties of undefined (reading 'map')`。

#### 根本原因
前后端接口格式不匹配：
- 后端将 `rows` 和 `columns` 包裹在 `result` 对象中
- 前端期望 `rows` 和 `columns` 在响应根级别
- `rows` 格式也不一致：后端返回对象数组，前端期望二维数组

#### 修复状态
- ✅ 后端代码已修复
- ✅ SQL 执行服务单元测试通过
- ⚠️ E2E 测试待验证（测试环境网络问题）

#### 修复内容
1. 修改 `QueryExecuteResponse` - 添加前端兼容字段
2. 修改 `QueryRunResponse` - 添加前端兼容字段
3. 修改 `execute_sql_endpoint` - 添加 rows 转换逻辑
4. 修改 `run_query_endpoint` - 添加 rows 转换逻辑

---

### ISSUE-002: LLM 服务调用超时 (已解决)

**严重级别**: High
**状态**: 待修复

#### 问题描述
点击"一键运行"或"生成 SQL"后，请求超时（30秒），显示"网络连接失败"错误。

#### 期望行为
- SQL 在 5-10 秒内生成完成
- 显示生成的 SQL 代码块
- 显示查询结果

#### 实际行为
- 请求超时（30s timeout）
- 显示网络错误提示
- 无 SQL 生成结果

#### 可能原因
1. 后端 LLM 服务网络连接问题
2. DashScope API Key 无效或额度不足
3. 后端 LLM 配置错误（模型名称、端点等）

#### 建议修复
1. 验证 DashScope API Key 有效性
2. 检查后端 LLM 配置（模型名称应为 qwen-turbo 或 qwen-max）
3. 检查网络连接
4. 添加请求超时重试机制
